# 实践

> 通过动手实践来掌握技能。

## 规则

1. **从简单开始**：先完成最简单的可用版本
2. **及时反馈**：尽快让代码/作品运行起来
3. **记录问题**：遇到困难要记录问题和解决方案
4. **复盘总结**：完成后复盘哪些做得好，哪些要改进
5. **迭代优化**：在现有基础上持续改进

## 经验

- 编程：先跑通 Hello World，再逐步添加功能
- 乐器：慢练比快练效果好，先求准再求快
- 运动：动作标准比重量/速度更重要
- 做项目：完成比完美更重要，先做出来再优化

## 方法

### 项目驱动学习

通过做项目来学习新技能。

1. 选定一个想做的项目
2. 列出需要的技术栈
3. 边学边做，遇到问题再查资料
4. 完成后重构代码，优化架构

### 模仿-改进-创新

先模仿优秀的作品，再逐步加入自己的想法。

1. 找到优秀的参考作品
2. 完全模仿做出来
3. 加入自己的改进
4. 最终形成自己的风格
