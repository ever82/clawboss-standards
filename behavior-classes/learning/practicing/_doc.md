# Practicing

> Master skills through hands-on practice.

## Rules

1. **Start Simple**: Complete the simplest working version first
2. **Immediate Feedback**: Get code/work running as soon as possible
3. **Record Problems**: Record problems and solutions when encountering difficulties
4. **Review and Summarize**: After completion, review what was done well and what needs improvement
5. **Iterate and Optimize**: Continuously improve based on existing work

## Experiences

- Programming: Get Hello World running first, then gradually add features
- Musical instruments: Slow practice is better than fast practice, accuracy before speed
- Sports: Proper form is more important than weight/speed
- Projects: Done is better than perfect, make it first then optimize

## Methods

### Project-Based Learning

Learn new skills by doing projects.

1. Select a project you want to do
2. List required tech stack
3. Learn while doing, look up references when encountering problems
4. Refactor code and optimize architecture after completion

### Imitate-Improve-Innovate

First imitate excellent works, then gradually add your own ideas.

1. Find excellent reference works
2. Completely imitate and reproduce them
3. Add your own improvements
4. Eventually form your own style
